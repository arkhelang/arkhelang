# ADR 0008: v0.2 scope

Date: 2026-07-18
Status: Accepted

## Context

v0.1 shipped complete against ADR 0004. The v0.2 backlog accumulated in the
design sketch is larger than one release should carry, and the lesson of
v0.1 stands: small cuts ship, ambitious cuts drift.

## Decision

v0.2 delivers four things:

1. **The OKF emitter.** The knowledge projection for LLM consumption
   (Open Knowledge Format bundles): one concept document per entity, link,
   and action type, generated from annotations and declarations, never
   hand-edited. This is the "what an agent should know" half of the IR's
   "what an agent may do".
2. **Synonyms as first-class annotations.** A `synonyms` annotation on
   entities and properties, carried into contracts and the OKF projection,
   so grounding can match the words people actually use. Prompted by early
   reviewer feedback on ontology authoring interfaces.
3. **Resolved types on effects.** Each effect entry in a contract inlines
   the destination property's type block (type, and values for enum and
   state destinations), removing the join every emitter currently
   reimplements. Already motivated in the v0.1 review record.
4. **The Rust port, begun.** A Rust implementation of the validator against
   the frozen v0.1 golden fixtures, in the reserved arkhelang and arkhe
   crates. Conformance is defined by the fixtures; the port ships when they
   pass, whether or not that lands inside the v0.2 window.

Deferred again, deliberately: ordering semantics, stdlib expansion, Cedar
and OpenFGA emitters, protocol emitters (MCP, OpenAI, ADK), OWL/RDF export,
imports resolution.

## Consequences

- Items 1 and 3 change contract and emitter output: new golden files, and
  the contract format gains fields (additive, no breaking change to v0.1
  consumers reading known fields).
- Item 2 is a language addition (annotation key with defined semantics),
  not a metamodel change; the schema is untouched.
- Item 4 establishes the multi-implementation conformance discipline the
  golden files were designed for.
